# 節點資料流總結

- Java 的資料流支援方式,功能強大又具`自定的彈性`

- 事實上,關於 I/O 方面的討論可以寫成一本書

- 一個無限制的鍊結機制讓你可混合一些類別來達成任何想要的 I/O 功能

- 可產生一些自己加強的資料類別,以便在一個輸入鍊或輸出鍊其中的任何地方做插入

- `I/O 奠定將來網路程式設計的基礎`

- 物件永續性與序列化觀念奠定將來 RMI (遠端方法呼叫 – Remote Method Invocation)`分散式運算基礎`
